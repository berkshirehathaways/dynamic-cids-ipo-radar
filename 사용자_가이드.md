# 사용자 가이드 (빠르게 잘 쓰는 법)

## 1) 나를 가장 잘 쓰는 프롬프트 방식
- 목적 + 제약 + 완료조건을 한 번에 주세요.
- 예시:
  - "Vercel 프론트/백엔드 배포까지 하고, 최종 URL과 검증 결과를 보여줘"
  - "로직 수정하고 실제 API 응답으로 전/후 비교해줘"

## 2) 작업 요청 템플릿
아래 형식으로 주면 정확도가 가장 높습니다.

```text
[목표]
무엇을 최종 완료해야 하는지

[제약]
사용할 스택, 금지사항, 보안 조건

[완료 조건]
성공 판정 기준(예: URL 접속, API 200, 데이터 건수)
```

## 3) 에러가 날 때 가장 빠른 전달법
- 에러 전문 그대로 붙여주세요.
- 실행한 명령어도 같이 주세요.
- 스크린샷이 있으면 함께 첨부하세요.

## 4) 배포 관련 핵심 포인트
- 프론트(Vercel)와 백엔드 API URL은 반드시 분리해서 확인합니다.
- `VITE_API_BASE_URL`은 "프론트 주소"가 아니라 "백엔드 공개 URL"입니다.
- Supabase 키는 절대 노출하지 말고, 노출 시 즉시 회전(재발급)하세요.

## 5) 내가 해주면 좋은 것들
- "끝까지 해줘"라고 하면 배포/검증/로그 확인까지 진행합니다.
- "왜 이렇게 됐는지"라고 하면 로직 근거와 수치까지 설명합니다.
- "다음엔 안 삽질하게"라고 하면 체크리스트를 문서로 남겨줍니다.
